## 系統的待實作事項 (很多喔)

### 從急迫到沒那麼急排序

- [v] 修正某些事件會在特定時段剛到達時馬上觸發的問題
- [v] 處理任務失敗情境
- [v] 狀態數值變動 (體力、壓力) 選項或事件
- [v] 加入消耗品類型的物品 (麵包)
- [v] 物品獲取選項或事件
- [v] 物品使用/拋棄選項 (MVP 不做裝備，關鍵物品無法被使用/拋棄)
- [v] 物品時限實裝
- [v] 檢視特定物品功能 (之後可能改成完整的item targeting)
- [v] 遭遇的數值變動展示改善
- [v] 地圖顯示改善，包含對於條件式路徑的進階顯示，還要大地圖的展現方法
- [ ] 場景物件 (Props) 結構
- [ ] 「觀察」和「檢查」系統
- [ ] 模板式物件 (字條、書本等等)
- [ ] 新遭遇 - 劇情
- [ ] 陣營關係 (對人與對陣營) (敵對、同盟、中立)
- [ ] 新條件: 某陣營聲望 / 某NPC好感度
- [ ] 任務拋棄(Abandon)、背叛(Ditch)、相斥等等情境
- [ ] 完成第四與第五條路線 (請參考docs\hidden\mvp_available_quests.md)
    - [ ] 角色的預設對話上下文變動+角色移動狀態
    - [ ] 實作凱恩、米爾多雷、奧蘭
    - [ ] 實作「枷鎖」任務
    - [ ] 實作有關礦工自治聯盟的事件與相關遭遇
    - [ ] 實作「凱奇的測試」任務線 (一到三)
    - [ ] 實作其他重要地區的事件與相關遭遇
- [ ] 完成預計要做的幾個事件、遭遇與任務
- [ ] 經驗系統，包含外部經驗以及技能數值經驗
- [ ] 瀕危與崩潰特效 (體力過低與壓力過高)
- [ ] (麻煩) DM動態賦予+/-數值變化 (GRANT, REVOKE訊號)
- [ ] (麻煩) DM動態物品獲取以及資源耗竭性 (也適用GRANT, REVOKE訊號)
- [ ] 音樂與音效控制器
